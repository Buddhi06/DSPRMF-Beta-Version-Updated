Activity 2: Phase 4 and 5: Threat and Vulnerability Identification

STRIDE for Security Threat Modeling

This framework proposes using the STRIDE threat modeling methodology to identify security threats associated with the assets linked to each stage of the AI model lifecycle (identified in TM phase 3).

STRIDE has been recommended and applied as a threat modeling approach for AI models by independent agencies such as European Union Agency for Cybersecurity (ENISA) . STRIDE categorises security threats into six groups, based on its acronym:

  1. Spoofing
  2. Tampering
  3. Repudiation
  4. Information Disclosure
  5. Denial of Service
  6. Elevation of Privilege

STRIDE threat modeling considers the following six categories of threats, with each threat compromising a specific security objective.

Threat Category Description Security Property
Spoofing The attacker impersonates or masquerades as another person, system, or process to gain unauthorised access or perform malicious actions. Authenticity
Tampering An attacker modifies the data, either in transit or at rest, to manipulate the outcomes or disrupt the operations. Integrity
Repudiation An attacker deletes information about an action or transaction in order to deny it ever took place, and the system lacks the ability to prove otherwise. Non-repudiation
Information Disclosure Sensitive information is stolen, accessed, or intercepted by unauthorised users either in transit, at rest, or during processing. Confidentiality
Denial of Service (DoS) An attempt to disrupt the availability of a system, application, or service, making it inaccessible to legitimate users. Availability
Elevation of Privilege This is similar to spoofing, but instead of taking the identity of another, the attacker elevates their own security level to an administrator. Authorisation

Threat Identification

Each asset listed in Step 3 of the TM process should be analysed in terms of the above threat categories to identify the threats that affect it. For each asset, you may systematically ask the following questions:

  1. How could an attacker spoof this asset?
  2. How could an attacker tamper this asset?
  3. Could an action involving this asset be repudiated?
  4. Could information be improperly disclosed from this asset?
  5. Could this asset be subjected to Denial of Service?
  6. Could an attacker achieve Elevation of Privilege (EoP) through this asset?

LINDDUN for Privacy Threat Modeling

This framework proposes using the LINDDUN threat modeling methodology to identify privacy threats associated with the assets linked to each stage of the AI model lifecycle (identified in TM phase 3). LINDDUN categorises privacy threats into seven groups, based on its acronym:

  1. Linking
  2. Identifying
  3. Non-Repudiation
  4. Detecting
  5. Data Disclosure
  6. Unawareness
  7. Non-Compliance
Threat Category Description
Linking Learning more about an individual or a group by associating data items or user actions.
Identifying Learning the identity of an individual.
Non-Repudiation Being able to attribute a claim, i.e., to know, having done, having said something, to an individual.
Detecting Deducing the involvement of an individual through observation.
Data Disclosure Excessively collecting, storing, processing or sharing personal data.
Unawareness Insufficiently informing, involving or empowering individuals in the processing of their personal data.
Non-Compliance The system deviates from security and data management best practices, standards and legislation, leading to the incomplete management of risk.

Threat identification

Each asset listed in Step 3 of the TM process should be analysed in terms of the above threat categories to identify the threats that affect it. In this step, the security threats that have been already identified in the security threat identification process were analysed in terms of the LINDDUN threat categories to determine in which ways the threats could affect privacy of data.

Several tools including the following can be used to perform STRIDE threat modelling:

Tool Description Link to download
Microsoft Threat Modeling Tool (TMT) Microsoft Threat Modeling Tool (TMT) is a free open-source security tool designed to help security professionals and developers to identify and mitigate potential security threats in software systems including AI models. It is based on the STRIDE threat modelling framework and provides a graphical interface to conduct threat modelling and helps in generating threat reports. Microsoft TMT
OWASP Threat Dragon OWASP Threat Dragon is a free open-source, cross-platform application for creating threat models. It supports several threat modelling frameworks including STRIDE. OWASP Threat Dragon
IriusRisk IriusRisk is an automated threat modelling platform designed to help developers, security teams, and enterprises identify and mitigate security threats in software including AI models. It supports several threat modelling frameworks including STRIDE. IriusRisk
ThreatModeler ThreatModeler is an automated that helps organisations to identify and mitigate security threats in software applications including AI models. supports several threat modelling frameworks including STRIDE. ThreatModeler
STRIDE GPT STRIDE GPT is an AI-powered threat modeling tool that leverages OpenAI's GPT models to generate threat models for a given application based on the STRIDE methodology. STRIDE GPT
